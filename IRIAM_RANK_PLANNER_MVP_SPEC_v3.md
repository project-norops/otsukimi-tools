# IRIAM Rank Planner --- MVP仕様書 v0.1

## 1. 目的

IRIAMライバーが、現在のコミュニティランク状態と今後の日別ランクスコア計画を入力し、最大3か月先までのランク推移・集計期間・スキップパス残数を自動シミュレーションできる非公式ツールを提供する。

本ツールは「実績管理」より「未来のランク計画・逆算」を主目的とする。

## 2. MVPの価値

-   現在状態を30秒程度で入力できる
-   日付をタップして `+1 / +2 / +4 / +6 / SKIP / 休み` を配置できる
-   3か月先までランク推移を自動再計算できる
-   スキップパス不足や計画矛盾を警告できる
-   作成した計画をカレンダー画像・ICSとして外部利用できる
-   ログインなしで無料利用できる

## 3. 対象外

MVPでは以下を実装しない。 - IRIAMアカウント連携 -
IRIAM画面の自動読取・スクリーンショットOCR -
応援ポイントやボーダーの予測 - デイリーランクスコアの自動取得 -
ライバーデータ分析 - 課金 - 認証 - DB保存 - 複数ユーザー共有編集 -
イベント管理の高度化

## 4. 公式仕様を前提とする計算ルール

2026-07-13確認時点のIRIAM公式ヘルプを基準とする。

### 4.1 ランク

順序：
`D → C1 → C2 → C3 → C4 → C5 → B1 → B2 → B3 → A1 → A2 → A3 → S1 → S2 → S3`

境界ランクでは、それ以上のアップ・ダウンを行わない。

### 4.2 集計期間

通常の集計期間は7日間。

IRIAM画面の「あとN日」は今日を含まない残日数表示として入力し、内部計算では：
`remainingCalculationDays = N + 1`

例： 「あと6日」→ 今日を含む計算対象日は7日。

### 4.3 ランクアップ

累計デイリーランクスコアが18以上に達した日の日次スコア確定後、即時ランクアップ。

ランクアップ時： - ランクを1段階上げる - 累計スコアを0にリセット -
新しい集計期間を開始する

### 4.4 ランクキープ

集計期間終了時の累計スコアが12〜17の場合、ランクキープ。

キープ時： - ランク維持 - 累計スコアを0にリセット -
新しい集計期間を開始する

### 4.5 ランクダウン

集計期間終了時の累計スコアが0〜11の場合、ランクダウン。

ダウン時： - ランクを1段階下げる - 累計スコアを0にリセット -
新しい集計期間を開始する

### 4.6 日別計画値

選択可能値： - `+1` - `+2` - `+4` - `+6` - `SKIP` - `休み` - `未設定`

`休み` と `未設定` は計画上のスコア0として扱う。
ただし表示上は区別する。

### 4.7 スキップパス

-   SKIP使用日はデイリーランクスコア集計をスキップ
-   集計期間終了日を1日延長
-   所持スキップパスを1枚消費
-   毎週月曜0時頃、前週に1日以上配信している条件を満たす場合1枚付与
-   最大所持数10枚

MVPでは実際の配信有無を自動取得できないため、月曜ごとの付与を以下のいずれかで扱う。

初期設定： `weeklySkipPassGrant = true`

ユーザーは各月曜の付与予定をON/OFF変更可能。

将来の仕様変更に備え、スキパ付与ロジックは独立したルール関数とする。


### 4.7.1 臨時スキップパス付与
通常の週次付与とは別に、任意の日付へ臨時のスキップパス付与を手動登録できる。

用途例：
- お盆
- 年末年始
- キャンペーン
- 運営による特別配布
- その他の臨時付与

入力：
- 付与日
- 付与枚数 `+1` 以上
- 任意メモ

計算ルール：
- 指定日の開始時点で付与として反映する
- 通常の月曜付与と同日に重なる場合は両方を加算する
- 加算後も所持上限10枚を超えない
- 上限超過分は切り捨てる
- ランクスコア計算や集計期間には直接影響しない
- 臨時付与予定を変更・削除した場合、3か月シミュレーション全体を再計算する

表示：
- カレンダー上に `スキパ +N` と表示する
- SKIP使用とは視覚的に区別する
- light green系の補助イベントとして表示する

実装上、通常の週次付与と臨時付与は別データとして保持し、
最終的な所持枚数計算時に統合する。

## 5. 初期入力画面

必須： - 基準日（default: 今日） - 現在ランク -
現在の累計デイリーランクスコア（0〜17を基本入力範囲） -
IRIAM画面表示「あとN日」 - 現在のスキップパス所持数（0〜10）

任意： - 計画名 - ライバー名または表示名

開始ボタン： `3か月の計画を作る`

## 6. メイン画面

### 6.1 カレンダー

月間カレンダーを縦方向に連続表示。 表示範囲は基準日から約3か月。

各日セル： - 日付 - 日別計画値 - メモ短縮表示 - ランク変動マーカー -
集計期間終了マーカー - スキパ残数警告

### 6.2 日付編集

日付タップでボトムシートを表示。

入力： - `+1 / +2 / +4 / +6 / SKIP / 休み / 未設定` -
メモ（任意、最大50文字） - 月曜日の場合：スキパ付与予定 ON/OFF
- 臨時スキパ付与枚数（任意、0以上）
- 臨時スキパ付与メモ（任意）

保存後、全期間を即時再計算する。

### 6.3 ランク推移表示

計算イベントをカレンダー上に表示。

例： - `S1 → S2 ランクアップ` - `S2 キープ` - `S2 → S1 ランクダウン`

集計期間は視覚的な帯または区切り線で表現する。



## 6.4 記念日自動表示
デビュー日が入力されている場合、計画期間内に該当する記念日を自動生成し、カレンダーに表示する。

対象記念日：
- デビュー1か月記念
- デビュー3か月記念
- デビュー6か月記念
- デビュー1周年
- 1周年以降は6か月ごとの節目
  - 1年6か月
  - 2周年
  - 2年6か月
  - 3周年
  - 以降同様
- デビュー100日記念
- 以降100日ごとの記念日
  - 200日
  - 300日
  - 400日
  - 以降同様

表示例：
- `1か月記念`
- `3か月記念`
- `半年記念`
- `1周年`
- `1年6か月記念`
- `100日記念`
- `500日記念`

計算ルール：
- 月・年ベースの記念日は暦日基準で計算する
- 100日単位の記念日はデビュー日を0日目として経過日数で計算する
- 同一日に複数の記念日が重なる場合は両方表示する
- 記念日はシミュレーション計算に影響しない表示イベントとして扱う
- ユーザーによる手入力・削除対象にはしない
- デビュー日を変更した場合、記念日一覧を再生成する

UI：
- 記念日はpale lavenderまたはsoft pink系のchip / eventとして表示
- ランク計画値より視覚優先度を下げる
- 日付タップ時に記念日の詳細を確認できる

## 7. 警告

以下を自動検知する。

### ERROR

-   スキップパス残数0でSKIPを配置
-   初期入力値が不正

### WARNING

-   集計期間終了時に0〜11見込み
-   ユーザーの計画上、意図せずランクダウンする
-   月曜スキパ付与OFFにより後続SKIPが不足する

警告は計算を停止しない。
不正なSKIPは「使用不能」として計算し、明示する。

## 15. UI・ビジュアルデザイン方針

### 15.1 基本コンセプト
スマホで日常的に開きたくなる、明るく可愛いパステル調のUIとする。

基調色：
- ピンク
- ライトグリーン
- 白

「可愛い」は装飾キャラクターではなく、配色・余白・角丸・タイポグラフィ・軽いアイコン表現で実現する。

### 15.2 禁止事項
- オリジナルキャラクターを作らない
- マスコットを配置しない
- IRIAM公式キャラクター・ロゴ・UI・背景を模倣しない
- 過剰なイラスト装飾を行わない
- 原色中心の強い配色にしない
- ゲーム画面風の派手な演出にしない

### 15.3 カラーパレット
初期候補：

- Main Pink: `#FF6B91`
- Soft Pink: `#FFE4EC`
- Pale Pink: `#FFF5F8`
- Light Green: `#DFF5E3`
- Accent Green: `#6BCB82`
- White: `#FFFFFF`
- Background: `#FFFDFE`
- Main Text: `#333333`
- Sub Text: `#777777`
- Border: `#F2DDE4`

色値はUI実装時に視認性を確認して微調整可能。
ただしピンク・ライトグリーン・白の基本構成は維持する。

### 15.4 UIスタイル
- card-based layout
- 大きめの角丸
- 細く淡いborder
- shadowは最小限
- 十分な余白
- スマホで押しやすいtap target
- 絵文字への過度な依存を避ける
- line iconまたはsimple iconを使用
- 重要情報のみ色を強くする

### 15.5 日別計画値の視覚表現
日別計画値は一目で識別できるchip/buttonとして表示する。

推奨：
- `+6`: strong pink
- `+4`: pink
- `+2`: soft pink
- `+1`: pale orangeまたはneutral accent
- `SKIP`: light green
- `休み`: light gray
- `イベント`: pale lavender
- `メモ`: pale yellow

色だけに依存せず、必ず文字ラベルも表示する。

### 15.6 ランク判定の視覚表現
- ランクアップ：pink accent
- キープ：light green
- ランクダウン見込み：warning tone

通常時は淡い色調を維持し、ランクダウン警告等の判断が必要な情報のみ強調する。

### 15.7 カレンダー
参考イメージの方向性：
- 月間カレンダー中心
- ランク集計期間を横長の帯で表示
- キープ期間はlight green
- ランクアップ予定・イベント等はpink系
- 日付セル内に `+6 / +4 / +2 / +1 / SKIP / 休み` と短いメモを表示
- 情報量が多い場合でも、文字の重なりを避ける
- 3か月分は縦スクロールを基本とする

### 15.8 PC表示
PCでは以下の3領域を基本候補とする。
- 左：navigation / plan actions
- 中央：calendar
- 右：current status / rank progress / skip-pass status / warnings

ただしMVPはmobile-firstとし、PC専用機能は追加しない。

### 15.9 モバイル表示
優先順位：
1. 現在ランク・累計スコア・残日数
2. カレンダー
3. 日別計画値入力
4. 警告
5. スキパ残数
6. export

スマホでは常時サイドバーを表示しない。
日付編集はbottom sheetを基本とする。

### 15.10 デザイン完成条件
- ピンク・ライトグリーン・白を基調としている
- キャラクター・マスコットを使用していない
- IRIAM公式デザインを模倣していない
- iPhone幅で主要操作が完結する
- 日別計画値を色と文字で識別できる
- ランクアップ / キープ / ダウン見込みを即座に判別できる
- 3か月表示でも計画を追える
- おつきみつーるずのトップページにツール一覧をカード表示できる
- 第1弾は利用可能、未開発ツールは「開発中」と明示される
- ライバー向け / リスナー向けのカテゴリーがメニュー上で明確に分離される
- 両対象のツールは共通metadataから両カテゴリーに正しく表示される
- アイコンリング装着ツールが「開発中」として表示される

## 16. おつきみつーるず構想

### 16.1 サイト位置付け
本MVPは単独サービスではなく、ライバーとリスナー向けの小型Webツール集「おつきみつーるず」の第1弾として公開する。

ブランド名：
`おつきみつーるず`

基本コンセプト：
「ライバーとリスナーのための、ちいさくて便利なツール置き場」

各ツールは原則として以下を重視する。
- 1つの困りごとを短時間で解決する
- スマホ優先
- 操作説明なしでも使いやすい
- 過剰な会員登録を求めない
- AI常時利用を前提にしない
- 小さく公開し、反応を見て改善する

本ランク計画ツールは「おつきみつーるず」内の1プロダクトとして扱う。

### 16.2 ポータル / トップページ
サイトトップでは、ツールを対象ユーザー別に明確に分けて表示する。

メニュー / 一覧カテゴリー：
- `ライバー向け`
- `リスナー向け`

PC・モバイルとも、この2カテゴリーを主要なナビゲーション単位とする。
同一画面に全ツールを無差別に並べない。

表示方法：
- トップページ上でカテゴリーごとにセクションを分ける
- ヘッダーまたは主要メニューから `ライバー向け` / `リスナー向け` へ移動できる
- モバイルではタブ、segmented control、または明確なカテゴリーボタンを使用してよい
- PCではサイドナビまたは上部ナビを使用してよい
- 実装方式はmobile-first usabilityを優先して自律判断する

対象が `ライバー向け / リスナー向け` のツールは、両カテゴリーに表示してよい。
カード定義は重複管理せず、単一のtool metadataからカテゴリー別にfilterして表示する。

各カード：
- ツール名
- 1行説明
- 対象：`ライバー向け` / `リスナー向け` / `ライバー向け / リスナー向け`
- status
- 利用可能な場合はツールへの導線

status：
- `available`：公開中
- `in_development`：開発中

表示ラベル：
- `available` → `使えます`
- `in_development` → `開発中`

`in_development` のカードも非表示にせず一覧へ表示する。
ただし、未実装ページへの壊れたリンクは作らない。
開発中カードはdisabled状態または詳細なしの静的表示とする。

### 16.3.1 ツール一覧データ構造

## 18. NOROPS / CiteRootとの完全分離

### 18.1 分離原則
「おつきみつーるず」は、NOROPS / CiteRootとは完全に別のプロジェクトとして扱う。

以下を共有しない。
- repository
- git history
- GitHub repository
- Vercel project
- Vercel project link
- deployment target
- environment variables
- domain設定
- project-specific config
- source code
- documentation
- issue / backlog管理

NOROPS / CiteRootのrepository、設定、deploy先、domain、Vercel projectを変更してはならない。

### 18.2 Repository boundary
おつきみつーるずのrepository rootは以下のみ。

`C:\dev\otsukimi-tools`

NOROPS / CiteRoot側のrepository pathへ移動・編集・commit・pushしてはならない。

特に以下は禁止する。
- `C:\dev\norops` 配下の変更
- NOROPS / CiteRoot repositoryへのfile copy
- NOROPS / CiteRootの`git remote`利用
- NOROPS / CiteRootのVercel projectへのdeploy
- NOROPS / CiteRootのenvironment variable参照・転用

### 18.3 GitHub
おつきみつーるず専用のGitHub repositoryを使用する。

推奨repository name：
`otsukimi-tools`

既存のNOROPS / CiteRoot repositoryをremoteとして設定してはならない。

push前に以下を確認する。
- current working directoryが`C:\dev\otsukimi-tools`
- git rootが`C:\dev\otsukimi-tools`
- `git remote -v` がNOROPS / CiteRoot repositoryを指していない

専用remoteが未設定の場合は、既存projectへの誤pushを避けるため停止し、最終報告で明示する。

### 18.4 Vercel
おつきみつーるず専用のVercel projectを使用する。

推奨project name：
`otsukimi-tools`

NOROPS / CiteRootのVercel projectへlink / deployしてはならない。

deploy前にVercel project linkを確認し、NOROPS / CiteRoot projectを指している場合はdeployを中止する。

`.vercel/project.json` が既存project由来の場合は再利用しない。

### 18.5 誤混在防止
Git / Vercel操作前に必ずproject boundary checkを実行する。

最低確認項目：
- `pwd` またはcurrent path
- `git rev-parse --show-toplevel`
- `git remote -v`
- Vercel project link情報

いずれかがNOROPS / CiteRootを指す場合、commit / push / deployを実行しない。



## 19. 中断・再開耐性

### 19.1 目的
Codexの利用上限、セッション中断、端末切断、コンテキスト圧縮等が発生しても、
作業状況を失わず未完了地点から再開できるようにする。

### 19.2 進捗ファイル
repository内に `docs/BUILD_PROGRESS.md` を作成する。

最低限記録する項目：
- Current status
- Completed
- In progress
- Next
- Tests
- Known issues
- Blockers
- Last updated

### 19.3 更新タイミング
少なくとも以下の完了時に更新する。
- scaffold完了
- rule engine完了
- simulator / tests完了
- portal完了
- rank calendar UI完了
- export完了
- quality check完了
- Git push完了
- Vercel preview deploy完了

重要な設計判断や未解決問題が発生した時点でも更新する。

### 19.4 再開手順
中断後は、実装再開前に必ず以下を読む。
1. `IRIAM_RANK_PLANNER_MVP_SPEC.md`
2. `CODEX_TASK_IRIAM_RANK_PLANNER_MVP.md`
3. `docs/BUILD_PROGRESS.md`

その後：
- repositoryの現在状態を確認
- 完了済み作業を再実装しない
- `In progress` / `Next` から再開
- 既存テスト状況を確認
- 必要に応じて再テスト

### 19.5 禁止事項
- セッション中断を理由に最初から作り直さない
- `BUILD_PROGRESS.md` とrepository状態が矛盾する場合に進捗ファイルだけを信用しない
- 未確認の完了項目を`Completed`へ記録しない

repositoryの実体とテスト結果を最終的な事実とする。

## 20. PWA / ホーム画面追加

### 20.1 目的
「おつきみつーるず」をスマートフォンのホーム画面へ追加し、Webサイトではなく日常的に使う軽量ツールとして起動しやすくする。

ネイティブアプリ化は行わない。MVPではPWAとして、ホーム画面追加とstandalone表示に対応する。

### 20.2 必須要件
- Web App Manifestを用意する
- `display: standalone` を基本とする
- app name / short nameを設定する
- theme color / background colorをブランド配色に合わせる
- ホーム画面用app iconを設定する
- iPhone / Android双方でホーム画面追加しやすい構成にする
- PWA起動時も主要routeが通常通り利用できる
- `/tools/rank-calendar` をPWA内で利用できる

推奨表示名：
- name: `おつきみつーるず`
- short_name: `おつきみ`

### 20.3 アイコン
MVPでは、ピンク・ライトグリーン・白を基調としたシンプルなapp iconを使用する。

禁止：
- IRIAM公式ロゴ・アイコンの模倣
- オリジナルキャラクター・マスコット
- 複雑な文字主体のアイコン
- NOROPS / CiteRootのブランド要素流用

アイコンは「おつきみつーるず」専用とする。

### 20.4 インストール導線
MVPではブラウザ標準のホーム画面追加機能を利用する。

必要に応じて、サイト内に控えめな案内を表示してよい。

例：
`ホーム画面に追加すると、アプリみたいにすぐ使えます`

ただし以下は禁止する。
- 全画面install modal
- 強制的なinstall誘導
- dismissしにくいbanner
- ツール利用を妨げる案内

### 20.5 オフライン対応
MVPでは高度なoffline-first対応を必須としない。

最低限：
- PWA設定がbuildを壊さない
- 通常のオンライン利用を優先する
- offline cacheのためだけに複雑なservice worker実装を追加しない

将来、利用実態を見て必要ならオフライン対応を検討する。

### 20.6 完成条件
- iPhone / Androidでホーム画面追加可能なPWA構成になっている
- ホーム画面からstandalone表示で起動できる
- おつきみつーるず専用app iconが表示される
- PWA起動後にportal / liver / listener / rank-calendarを利用できる
- PWA設定によるtest / typecheck / lint / build failureがない
